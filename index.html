Architectural Migration

Nixie was originally conceptualized, built, and deployed as a monolithic R/Shiny application. As operational demands grew, I orchestrated the complete architectural migration to the current Go (Golang) / Wails compiled executable to guarantee extreme path-security, offline capabilities, and zero-footprint data retention.

Here are excerpts from my core operational guardrails given to the AI:

System.Rule: Anti-Auto-Pilot Protocol

AI scaling risks autonomous damage. I enforce a strict "Zero Trust" execution halt.

1. Default State: PAUSE. Do not call OS tools immediately. 2. Execute Trigger: You are ONLY allowed to execute code if the user explicitly includes: "RUN" or "EXECUTE". 3. Exceptions: Search/Read tools allowed for context gathering.

System.Rule: Empirical Troubleshooting

Preventing AI "hallucinations" during complex forensic bug hunting.

1. Environment: Cache reset? Correct server? 2. Syntax: V8 Console errors? 3. Logic: Is the execution flow firing? 4. Architecture: Is the core foundational design fundamentally flawed? (Never guess. Use the local Ryggraden agent to verify logic paths.)

Security Manifest (Iron Dome)

Because this system handles sensitive police data, the AI was strictly tethered to this architectural manifest during generation:

Path Access Controls (LFI Mitigation)

Strict runtime whitelisting forces all I/O exclusively into configured root directories. Operating system cores (system32) are statically blacklisted to prevent traversal vulnerabilities.

Volatile Memory Processing

All JSON/XML parsing and entity extractions occur in volatile RAM. Zero unencrypted temp-files are flushed to disk during the correlation phase.

Execution Depth Capping (Zip-bomb proof)

The recursive traversal engine is hard-capped at 250 objects per case folder. This prevents resource exhaustion attacks from deeply nested or cyclical folder payloads.

Total Offline App Isolation

The Wails framework encapsulates the system without exposing localhost ports. Inter-process communication operates over local memory bindings only.

Systems Architecture

Go Backend (Data / I/O) ⟷ Wails IPC (Bridge) ⟷ TypeScript / Vite (Frontend GUI) ⟷ SQLite DB (State)
